# 错误处理与错误码

---

## 1. 三级处理策略

```
操作执行
  │
  ├─ 成功 → 返回结果
  │
  └─ 失败 → [1] 自动重试（最多 N 次）
              │
              ├─ 成功 → 返回结果（对用户透明）
              │
              └─ 仍失败 → [2] 降级运行
                            │
                            ├─ 有降级方案 → 继续，标记降级状态
                            │
                            └─ 无降级方案 → [3] 终止，返回错误码
```

| 级别 | 行为 | 用户感知 |
|------|------|---------|
| 重试 | 自动重试 3 次，间隔递增（1s/3s/5s） | 无感知 |
| 降级 | 关闭受影响功能，其余继续 | 功能受限提示 |
| 终止 | 返回错误码，记录日志 | 错误提示 + 错误码 |

---

## 2. 各组件降级策略

| 组件 | 失败场景 | 重试 | 降级方案 | 终止条件 |
|------|---------|------|---------|---------|
| 编码器 | embedding 计算超时/失败 | 3 次 | 跳过该对象，标记 encoding_failed | 超过 10% 失败则终止 |
| 模型训练 | GPU OOM / 收敛失败 | 1 次（减 batch） | 降为 CPU 训练 / 跳过本轮训练用上轮模型 | 两轮连续失败 |
| 模型推理 | 内存不足 | 1 次（分批） | 关闭机器作答，纯人类标注 | 不影响主流程 |
| 弱监督 | LF 执行超时 | 1 次 | 关闭弱监督，退化为纯 SALT | — |
| 覆盖率计算 | 数值溢出 | 1 次 | 用上轮覆盖率近似，标记 inaccurate | — |
| 标注业务 | 数据库死锁 | 3 次 | — | 返回错误，前端提示刷新 |
| 导出 | 文件过大 | 1 次（分片） | — | 返回分片下载链接 |

## 3. 错误码设计

格式：`{模块}-{子模块}{序号}`，三段式，易于定位。

### 3.1 通用 (A0)

| 错误码 | 含义 |
|--------|------|
| A0-001 | 参数校验失败 |
| A0-002 | 未登录 |
| A0-003 | 无权限 |
| A0-004 | 资源不存在 |
| A0-005 | 请求频率超限 |
| A0-006 | 数据库错误 |
| A0-007 | 内部未知错误 |

### 3.2 项目 (PJ)

| 错误码 | 含义 |
|--------|------|
| PJ-001 | 项目状态不允许此操作（如 COMPLETED 项目不可启动） |
| PJ-002 | 标签集已锁定不可修改 |
| PJ-003 | 免费版项目数已达上限 |
| PJ-004 | 免费版样本数已达上限 |
| PJ-005 | 免费版不支持删除项目 |
| PJ-006 | 导入格式不支持 |
| PJ-007 | 字段映射不完整（required 字段未映射） |
| PJ-008 | 算法插件未找到 |

### 3.3 标注任务 (TK)

| 错误码 | 含义 |
|--------|------|
| TK-001 | 无任务可抢（池空） |
| TK-002 | 持有数已达上限 |
| TK-003 | 任务已被他人提交（乐观锁冲突） |
| TK-004 | 任务不属于当前用户 |
| TK-005 | 任务状态不允许此操作 |
| TK-006 | 同一标记对象已答过 |

### 3.4 模型 / 算法 (MD)

| 错误码 | 含义 |
|--------|------|
| MD-001 | 编码器不可用 |
| MD-002 | embedding 计算失败（批量） |
| MD-003 | 模型训练失败 |
| MD-004 | 模型推理失败 |
| MD-005 | 覆盖率计算异常 |
| MD-006 | 弱监督引擎不可用 |
| MD-007 | 训练收敛过慢（loss 不下降） |
| MD-008 | GPU 资源不足 |

### 3.5 导入导出 (IO)

| 错误码 | 含义 |
|--------|------|
| IO-001 | 文件格式错误 |
| IO-002 | 文件过大 |
| IO-003 | 编码不支持 |
| IO-004 | 导出数据为空 |
| IO-005 | CSV 表头缺失或格式不符 |

### 3.6 用户 / 权限 (UR)

| 错误码 | 含义 |
|--------|------|
| UR-001 | 用户名已存在 |
| UR-002 | 密码不符合要求 |
| UR-003 | 内置角色不可删除 |
| UR-004 | 项目不可见（PRIVATE 且非成员） |
| UR-005 | 项目内角色无此操作权限 |

---

## 4. 错误响应格式

```json
{
  "error": {
    "code": "TK-003",
    "message": "任务已被他人提交",
    "detail": "anno_id=a1, version mismatch: expected 3, got 4",
    "retryable": false
  }
}
```

| 字段 | 说明 |
|------|------|
| code | 错误码，用户反馈给技术支持 |
| message | 人类可读描述，前端可直接展示 |
| detail | 技术细节，仅开发/调试可见 |
| retryable | 前端是否可自动重试（如 TK-001 可轮询重试） |

---

## 5. 前端降级展示

| 场景 | 展示 |
|------|------|
| 重试成功 | 无提示（透明） |
| 降级 | 页面顶部黄色横幅："编码器暂时不可用，部分对象跳过，已自动重试。错误码 MD-002" |
| 终止 | 红色弹窗："模型训练失败，请稍后重试或联系技术支持。错误码 MD-003" |
| 标注冲突 | toast 提示 + 自动刷新，不弹窗阻断 |
