# 授权与版本模型

私有部署场景，通过 license 文件区分免费版和订阅版。

---

## 1. 版本对比

| | Free | Pro（订阅） |
|------|------|------|
| 样本总数（全部项目合计） | ≤ 200 条 | 无限制 |
| 项目数上限 | ≤ 10 个 | 无限制 |
| 删除项目 | 不支持 | 支持 |
| 用户数 | 不限 | 不限 |
| 功能完整性 | 完整 | 完整 |
| 有效期 | 永久 | 按订阅周期 |
| 过期后行为 | — | 锁定标注和创建，保留查看/导出 |
| 技术支持 | GitHub Issues | 邮件/工单 |

---

## 2. license 文件

### 结构

```json
{
  "licensee": "XX公司",
  "tier": "free",
  "issued_at": "2026-01-01T00:00:00Z",
  "expires_at": null,
  "limits": {
    "max_samples": 200,
    "max_projects": 10,
    "allow_project_delete": false
  },
  "features": ["*"],
  "signature": "base64-encoded-signature"
}
```

- `tier = "pro"` 时，`limits` 下所有字段为 `null`，即无限制
- `signature` 由你的私钥签发，服务端启动时用内置公钥验证

### 放置位置

```
{部署目录}/license.key
```

启动时读取，校验失败 → 拒绝启动。

---

## 3. 免费版限制生效点

| 操作 | 校验逻辑 |
|------|---------|
| 导入数据 | 若 `已导入总数 + 本次导入 > 200`，截断至恰好 200 条，丢弃超出部分，提示用户 |
| 创建项目 | 若 `已有项目数 ≥ 10`，拒绝创建 |
| 删除项目 | 直接拒绝，返回 "免费版不支持删除项目" |
| 已超额时创建项目 | 若 `样本总数已达 200`，仍可创建新项目但无法导入（因为没额度了） |

### 导入截断提示

```
提示：免费版上限 200 条样本，已导入前 200 条，其余已忽略。
升级 Pro 版以解除限制。
```

---

## 4. Pro 版过期降级

订阅到期后：

| 操作 | 行为 |
|------|------|
| 登录、浏览 | 正常 |
| 查看已有项目、导出已有结果 | 正常（数据不丢） |
| 创建新项目 | 拒绝 |
| 导入数据 | 拒绝 |
| 标注（抢单/提交） | 拒绝 |
| 审阅 | 拒绝 |
| 删除项目 | 拒绝 |

页面顶部显示横幅："授权已过期，标注功能已锁定。查看和导出历史数据不受影响。请续期。"

---

## 5. license 校验

不做重型反破解——AI 时代代码在客户机器上，防不住。轻量校验足以区分"合规客户"和"未授权使用"：

- license 文件带签名，本地用公钥验证，防止随意篡改过期时间
- 不做在线授时校验（无需联网）
- Pro 版过期后功能锁定，仅保留查看和导出

付费的真正价值不在技术封锁，而在**持续更新和技术支持**。

---

## 6. Pro 版交付内容

| 项目 | Free | Pro |
|------|------|-----|
| 软件本体 | ✓ | ✓ |
| 功能限制 | 200 条 / 10 项目 / 不可删除 | 无限制 |
| 版本更新 | 安全补丁 | 全部新功能 |
| 技术支持 | GitHub Issues | 邮件/即时通讯/远程协助 |
| SLA | 无 | 约定响应时间 |
| license 有效期 | 永久 | 按订阅周期，过期后可续 |

---

## 7. 签发流程

```
客户购买 Pro 订阅
  → 在授权管理后台填写：客户名称、订阅时长（月/年）
  → 系统生成 license 文件，用私钥签名
  → 客户下载 license.key 放入部署目录，重启生效
```

续期：延长 `expires_at`，重新签发即可。
