# 弱监督术语表

---

## 核心概念

| 术语 | 英文 | 说明 |
|------|------|------|
| **标注函数** | Labeling Function (LF) | 一条自动标注规则。输入标记对象，输出标签或弃权。由管理员编写，可跨项目复用 |
| **弃权** | Abstain | LF 无法判断时返回空（`None` 或 `⊥`），表示"这条规则不适用于该对象" |
| **弱标签矩阵** | Weak Label Matrix \( \Lambda \) | 论文概念。实践中不存矩阵，每条 LF 直接对文本执行，输出即消费。最终只落盘软标签/硬标签/置信度 |

| **融合模型** | Label Model (P) | 轻量概率图模型（非深度学习）。从 LF 间的一致/矛盾模式推断每条 LF 的可靠性，综合给出软标签。需要完整 Λ 矩阵（不能逐条）。权重每次重新估计，不固化 |
| **软标签** | Soft Label | 融合模型输出的概率分布，每个类别一个概率值。内部中间产物，不出弱监督模块 |
| **硬标签** | Hard Label | 软标签经 LRR 修正后 argmax 取最终标签。给下游的唯一输出（或 null 表示被丢弃） |
| **置信度** | Confidence | 软标签中最大概率值，如 0.47。越高越确定。用于驱动人力分配 |
| **标签修正规则** | Label Refining Rule (LRR) | 用逻辑规则纠正 LF 输出的冲突或错误。形式为"若条件成立则标签 = w"或"若条件成立则丢弃" |
| **谓词** | Predicate | LRR 条件部分的原子判断，有四种：工具谓词（LF 的输出）、关系谓词（样本间相似）、软标签谓词（置信度阈值）、样本谓词 |
| **规则挖掘** | Rule Mining | 从历史标注数据中自动发现有效的 LRR |
| **衰减率** | Decay Rate \( \rho \) | 训练时弱标签权重的衰减速度。早期权重高（帮模型启动），后期趋零（让位于人工标签） |

---

## LF 集合 vs LRR 规则集

| 概念 | 说明 | 谁维护 | 复用 |
|------|------|--------|------|
| LF 集合 | 一组 labeling function，如"新闻分类规则 v1" | 管理员 | 多个项目可共用 |
| LRR 规则集 | 一组标签修正规则，如"新闻裁决规则" | 管理员 / 自动挖掘 | 多个项目可共用 |

## 一个 LF 的组成

```
名称: LF_keyword_sports
描述: 体育类关键词匹配
代码: def LF(text): return "体育" if any(kw in text for kw in [...]) else None
```

- 输入：标记对象（由 anno_type 的 model 视图拼装的文本）
- 输出：标签值（字符串）或 `None`（弃权）
- 语言：Python
