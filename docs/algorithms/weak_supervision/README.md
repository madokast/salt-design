# 弱监督

论文第 6 章。全自动，不需专家参与。管理员写好 labeling functions 和 LRR 规则后，系统自动生成弱标签辅助模型训练。

纯计算，不操作数据库。上层采集数据传入，算法返回结果，上层负责入库。

---

## 调用方职责

```
上层:
  1. 采集: 本轮选中对象、所有 LF 定义、LRR 规则集、已有人工标注（校准用）
  2. 调用算法: weak_labels = generate(objects, LFs, rules, human_labels)
  3. 入库: 用修正后的弱标签参与模型训练（加权）
```

---

## 两步流程

```
标记对象 × N
      │
      ▼
┌──────────────────┐
│ 1. LF 执行（可逐条） │  ← 每条 LF 独立处理文本，不需要看别的对象
│   输出完整 Λ 矩阵     │
└────────┬─────────┘
         │ 必须全部 LF 全部对象执行完
         ▼
┌──────────────────┐
│ 2. 融合模型 P（批量） │  ← 从行间一致/矛盾模式推断 LF 可靠性
│   输出软标签         │     必须拿到完整 Λ，不能逐条
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 3. LRR 修正（可逐条）│  ← 逻辑规则匹配，不依赖其他行
└────────┬─────────┘
         │
         ▼
  硬标签 + 置信度（最终输出）
```

融合模型 P 不是深度学习，是一个轻量概率图模型（类似 Naive Bayes 变体），用 EM 或 Gibbs 采样从 LF 间的一致/矛盾模式推断可靠性。权重每次执行重新估计，不固化。

软标签是内部中间产物，不出模块。最终输出给下游的是硬标签（或 null）+ 置信度。

---

## 第一步：融合模型 P

### 输入

- 弱标签矩阵 \( \Lambda \): \( n \) 个对象 × \( m \) 条 LF，每个元素为标签值（1~C）或 \( \perp \)（弃权）
- 人工标注：少量已标注对象的标签（用于校准）

### 思路

每条 LF 是一个有噪声的标注源。P 推断每条 LF 的类别条件错误率（如"LF_1 在真实标签=正面时输出负面的概率是 12%"），然后综合所有 LF 给出每个对象的软标签。

### 输出

每个对象每个类别的概率 \( Pr(x, w) \)，\( w \in \{1, ..., C\} \)。

---

## 第二步：LRR 规则修正

### 规则形式

```
φ: X → x.label = w
```

X 是谓词的合取。谓词类型：

| 谓词 | 含义 | 示例 |
|------|------|------|
| `T(LF, x, w)` | LF 对 x 输出 w | LF_keyword 输出 sports |
| `¬T(LF, x, w)` | LF 对 x 不输出 w | LF_regex 没有输出 business |
| `M(x, x')` | 两样本满足某种关系 | 同一主题 |
| `Pr(x, w) ⊗ c` | 融合模型置信度比较 | Pr(x, sports) < 0.1 |

### 规则类型

| 类型 | 特征 | 效果 |
|------|------|------|
| 冲突裁决 | 多条 LF 矛盾 | 选一条更可靠的 |
| 证据传播 | 含 M(x, x') 谓词 | 从已确定样本向关联样本传播标签 |
| 噪声过滤 | 后果为 ⊥ | 丢弃不可靠的弱标签 |

### 规则挖掘（离线）

从历史标注数据中挖掘 LRR。水平搜索谓词空间，用自适应剪枝控制搜索代价。规则按置信度和覆盖率排序，低质量丢弃。

### 规则执行（在线）

对每条对象：
1. 匹配所有适用的 LRR
2. 若多条规则冲突 → 按规则置信度 + 特异性 + 与 P 的一致性加权排序，取最高
3. 若最高两条得分太接近 → 弃权（保守）
4. 应用选中规则的后果：确认标签 / 修正标签 / 丢弃标签

---

## 输出

给下游的是硬标签（或空）+ 置信度。软标签只在模块内部流转，用于 LRR 的阈值判断。

```
WeakLabelResult:
  labels: [{anno_id, label: "体育" | null, confidence: 0.47}, ...]
  discarded: int              # 被丢弃的数量（label = null）
  corrected: int              # 被 LRR 修正的数量
  coverage: float             # 弱标签覆盖率
  accuracy: float | null      # 准确率（有人工标注校准时）
```

---

## 在训练中的使用

弱标签和人工标签混合训练，权重随 epoch 衰减：

\[
\mathcal{L}(t) = e^{-\rho t} \sum_{(x,\tilde{y})\in D_W} \ell(x,\tilde{y}) + (1-e^{-\rho t}) \sum_{(x,y)\in D_S} \delta_x \ell(x,y)
\]

- \( t \): 训练 epoch
- \( \rho \): 衰减率（默认 0.5）
- \( \delta_x \): 覆盖半径，给覆盖范围大的标注点更高权重

早期弱标签权重高（样本少时帮模型起步），后期人工标签权重高（保证精度）。

## 完整示例

以"新闻四分类"项目为例，标签集 = {财经, 体育, 科技, 娱乐}。

### 管理员配置的 LFs

```python
def LF_keyword_money(text):
    """出现财经关键词 → 财经"""
    keywords = ["股价", "涨停", "央行", "GDP", "A股"]
    for kw in keywords:
        if kw in text:
            return "财经"
    return None  # 弃权

def LF_keyword_sports(text):
    """出现体育关键词 → 体育"""
    keywords = ["NBA", "世界杯", "决赛", "联赛", "冠军"]
    for kw in keywords:
        if kw in text:
            return "体育"
    return None

def LF_regex_stock(text):
    """匹配股票代码 → 财经"""
    import re
    if re.search(r'\d{6}\.(SZ|SH)', text):
        return "财经"
    return None

def LF_entity_org(text):
    """包含知名公司 → 科技"""
    companies = ["苹果", "华为", "谷歌", "微软", "特斯拉"]
    for c in companies:
        if c in text:
            return "科技"
    return None
```

### 一条待标注文本

> "苹果公司今天发布财报，股价大涨5%，市值突破3万亿美元，分析师看好其AI战略。NBA方面，湖人队晋级季后赛下一轮。"

### LF 输出

| LF | 匹配依据 | 输出 |
|----|---------|------|
| LF_keyword_money | "股价" | 财经 |
| LF_keyword_sports | "NBA" | 体育 |
| LF_regex_stock | 无股票代码 | 弃权 |
| LF_entity_org | "苹果" | 科技 |

**冲突：三条 LF 给出三个不同标签。**

---

### 融合模型 P 的处理

P 不直接投票。它从历史数据中学习了每条 LF 的可靠性：

| LF | 推断准确率 | 对"财经"的倾向 | 对"体育"的倾向 |
|----|-----------|---------------|---------------|
| LF_keyword_money | 78% | 强 | — |
| LF_keyword_sports | 91% | — | 强 |
| LF_entity_org | 62% | — | — |

P 综合计算，输出软标签：

```
Pr(财经) = 0.42
Pr(体育) = 0.47
Pr(科技) = 0.09
Pr(娱乐) = 0.02
```

最可能是体育（0.47），但信心不高——因为各 LF 确实矛盾。

---

### LRR 规则介入

管理员配置了以下 LRR（从历史数据挖掘）：

```
规则 1: 若 LF_keyword_sports 输出体育 且 LF_entity_org 输出科技
        → 最终取体育
        （历史数据中，体育关键词比公司名更可靠）

规则 2: 若 Pr(x, 标签) < 0.15 且无其他 LF 支持
        → 丢弃该标签
```

#### 规则 1 执行

- 前提条件：LF_keyword_sports=体育 ∧ LF_entity_org=科技 → 满足
- 后果：x.label = 体育
- 效果：科技直接被否决，剩下财经 vs 体育

#### 规则 2 执行

- 前提条件：Pr(科技) = 0.09 < 0.15，且只有 entity_org 支持科技 → 满足
- 后果：丢弃科技标签
- 这与规则 1 效果一致，双重保证

#### 最终结果

LRR 修正后：标签 = 体育，confidence = 0.47

---

### 另一种情况：全一致

> "央行宣布降准，A股三大指数集体收涨"

| LF | 输出 |
|----|------|
| LF_keyword_money | 财经 |
| LF_regex_stock | 弃权 |
| LF_entity_org | 弃权 |

只有一条 LF 生效，且融合模型置信度高（Pr(财经) = 0.89）。无冲突，LRR 不触发。

最终结果：标签 = 财经，confidence = 0.89。

---

### 另一种情况：全部弃权

> "今天天气真好，适合出去玩"

所有 LF 都不命中 → 全部弃权 → 不生成弱标签。该对象留给人标或模型预测。

---

### 汇总

```
标记对象: "苹果公司今天发布财报..." 
  原始 LF 输出:  财经 / 体育 / 弃权 / 科技  (冲突)
  P 软标签:      财经(0.42) 体育(0.47) 科技(0.09) 娱乐(0.02)
  LRR:           规则1 触发 → 取体育
                规则2 触发 → 丢弃科技
  最终弱标签:     体育 (confidence=0.47)

标记对象: "央行宣布降准..."
  原始 LF 输出:  财经 / 弃权 / 弃权 / 弃权  (无冲突)
  P 软标签:      财经(0.89) ...
  LRR:           无规则触发
  最终弱标签:     财经 (confidence=0.89)

标记对象: "今天天气真好..."
  原始 LF 输出:  弃权 / 弃权 / 弃权 / 弃权
  最终弱标签:     无
```

弱标签覆盖率 = 2/3 = 66.7%，其中一条有冲突但被 LRR 成功裁决。

## 内置 vs 用户可配置

### 内置（系统提供，不可修改）

| 组件 | 说明 |
|------|------|
| 融合模型 P | Snorkel 风格概率模型，推断 LF 可靠性并输出软标签 |
| LRR 规则引擎 | 规则匹配、冲突排序、标签修正的执行器 |
| 规则挖掘器 | 从历史标注数据中自动挖掘 LRR，水平搜索 + 自适应剪枝 |
| 加权训练 | 弱标签和人工标签按 epoch 衰减混合训练 |

### 用户可配置（跨项目复用）

| 配置项 | 说明 | 存储 |
|--------|------|------|
| LF 集合 | 多条 labeling function，命名为"新闻分类规则 v1" | 全局 `lf_sets` 表 |
| LRR 规则集 | 多条逻辑修正规则，命名为"新闻裁决规则" | 全局 `lrr_sets` 表 |
| 衰减率 \( \rho \) | 弱标签权重衰减速度 | 项目级配置 |

### 项目关联

```
Project:
  lf_set_id: UUID | null        # 使用的 LF 集合
  lrr_set_id: UUID | null       # 使用的 LRR 规则集
  rho: float = 0.5              # 衰减率
```

LF 集合和 LRR 规则集是全局资源，多项目可共用同一套。

### 项目克隆时

克隆界面中，若源项目启用了弱监督：

```
☑ 算法名称 (salt_weak)
☐ 复用 LF 集合: "新闻分类规则 v1"    ← 默认指向同一套
☐ 复用 LRR 规则集: "新闻裁决规则"     ← 默认指向同一套
```

默认复用——同类任务同一套规则。若新项目需要不同规则，取消勾选后另行选择。

## 数据落盘

| 数据 | 落盘？ | 说明 |
|------|--------|------|
| LF 定义 | ✓ | `labeling_functions` 表，管理员配置 |
| LRR 规则 | ✓ | `lrr_rules` 表，管理员配置或自动挖掘 |
| 原始 LF 输出（矩阵） | ✗ | 每条 LF 实时执行，输出即消费。不存中间矩阵 |
| 融合模型 P 的软标签 | ✗ | 计算后直接传给 LRR，不落盘 |
| 最终弱标签 + 置信度 | ✓ | 写入 `anno_task` 的 MACHINE 行或独立弱标签字段 |
| 弱标签统计 | ✓ | coverage / accuracy / discarded / corrected，写入轮次记录 |
