# embedding 计算

---

## 定位

将标记对象转为定长浮点向量。所有后续算法（选样、覆盖分析、模型训练）都基于 embedding 空间进行。

---

## 当前版本（v1）：纯文本

- 输入：标记对象的文本字段（由 anno_type 的 model 视图拼接）
- 编码器：Qwen text embedding v4（或其他开源 SOTA 文本 embedding 模型）
- 输出：768 维浮点向量
- 批量调用，导入后一次性计算入库

```
原始文本 → encoder.encode() → [0.12, -0.34, 0.78, ...] (768维) → 入库
```

---

## 后续扩展：多模态管道

非文本标记对象（图片、视频、富文本），先经模态特定的预处理转为纯文本，再送入同一个文本编码器。

```
图片   → 图片描述模型 → "一只猫坐在沙发上"        ↘
视频   → 抽帧+描述    → "一个人在跑步..."         → encoder → embedding
OCR    → 文字识别     → "发票号: 123456"         ↗
富文本 → 标签剥离     → "这是一段加粗的重要文字"   ↗
```

### 图片

- 图片描述模型（BLIP-2 / LLaVA / Qwen-VL）生成自然语言描述
- 若图片中有文字，先 OCR（PaddleOCR / Tesseract）提取，拼入描述
- 最终进入文本编码器

### 视频

- 等间隔抽帧（如每秒 1 帧）
- 每帧经图片描述模型生成描述
- 可选：语音识别提取音频文字
- 多帧描述拼接为一段文本，进入文本编码器

### 富文本 / HTML

- 保留纯文本内容，去掉 HTML 标签、样式、脚本
- 可选：保留结构信息（标题层级、列表缩进）作为额外 token
- 进入文本编码器

### OCR 文档

- 扫描件 / PDF 先经 OCR 提取文字
- 可选：保留版面位置信息（坐标），拼入文本
- 进入文本编码器

## 统一管道

```
                        ┌─ 文本 ───────────→ 直接
                        │
标记对象 → 判断模态 → ──┼─ 图片 → caption → 文本
                        │            OCR    ↗
                        │
                        ├─ 视频 → 抽帧 → caption → 文本
                        │         语音识别 ↗
                        │
                        ├─ 富文本 → 标签剥离 → 文本
                        │
                        └─ OCR文档 → 识别 → 文本

                                               │
                                               ▼
                                    文本 encoder → embedding
```

所有模态最终汇入同一个文本编码器。好处：embedding 空间统一，后续 MLP 分类器无需感知模态差异。

---

## 编码器管理

- 内置若干开源编码器（Qwen v4、text2vec-large-chinese 等）
- 管理员可注册新编码器
- 编码器选定后全项目统一，更换编码器需全量重算
- 向量维度由编码器决定，下游 MLP 输入维度自动适配

---

## 批量调用

- 导入时全量批量调用，分 batch（如 256 条/batch）
- 失败自动重试 3 次，单条失败跳过并标记
- 超过 10% 失败则终止导入

---

## 存储

- embedding 存入标记对象表（`anno_content_{pid}.embedding`）
- 只算一次，后续轮次直接读库
