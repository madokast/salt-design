# SALT 标注平台

基于 [SALT 论文](./salt.md) 的企业级数据标注平台。核心思路：人只标一小部分，机器学完后标剩下的，且有数学理论保证自动标注精度。

---

## 功能概述

### 项目管理

管理员创建一个标注项目，设定标签集（如"正面、负面、中性"），这一定下来就不可修改。然后选择算法——内置"SALT 标准版"和"SALT + 弱监督"两种，后续可注册自定义算法。配置精度容忍度（愿意接受多少精度损失来换取省人工）、每轮标注量、是否跳过审阅阶段。上传待标注数据后，系统自动计算文本向量，向量化完成后即可启动标注流程。项目有明确的状态流转：草稿、向量化中、就绪、运行中、已完成，任何时候管理员可暂停项目。项目是一次性的，数据导入后不可追加。支持从已有项目克隆创建新项目，可选择克隆标签集、标记类型、项目成员、项目配置、算法名称；如果算法名不变还可复用已训练的模型，新项目从热启动开始，省掉冷启动轮次。

### 标注协作

多个专家同时在线抢单。页面顶部实时显示池子状态：待抢数量、自己持有几条、总共多少条，定时自动刷新。专家点击开始标注，系统从事务中随机取一条标记对象，渲染成 HTML 页面展示给专家。专家选标签后点击提交并抢下一条，两个操作在一个数据库事务中原子完成。如果池中已无待抢任务，但其他专家持有任务未提交，可直接抢过来——不管对方是否超时，先提交者生效。后来提交的人会看到"已被他人作答"的提示，页面自动刷新。每位专家同时持有的任务数有上限，防止一人囤积阻塞他人。专家可主动放弃手中任务归还池中。页面底部始终显示当前持有的任务列表摘要，可点击切换标注或放弃。

### 机器辅助

启用机器作答后，每轮模型先对样本做出预测，写入机器任务行。随后专家开始标注，标注界面上可选择显示或隐藏机器推荐，供专家参考。机器对每个样本给出置信度——预测很确定时只分配一个专家复核，预测摇摆不定时分配两个专家交叉验证。管理员还可以配置随机抽检比例，随机抽取一部分样本强制双人标注，用于持续监控专家标注质量，但不影响主流程推进。

### 多人标注与裁定

同一个标记对象可以有多个标注任务——可能是两个人类专家各标一次，也可能是一个人类加一个机器。当所有任务完成、标签一致时，系统自动回写最终结果。不一致时分情况处理：如果只是人与机器不一致且配置为信任专家，则直接采纳专家意见跳过裁定；其余情况（专家之间不一致、或配置为需裁定的人机不一致）进入管理员裁定列表。裁定界面同时展示所有标注者的标签、机器置信度、标记对象的完整渲染内容，管理员选定最终标签后回写。

### 审阅

每轮专家标注全部提交后，默认进入审阅阶段。管理员或审阅员逐条浏览本轮标注结果，可直接通过、修改标签后通过、或打回任务池。支持按专家筛选、按标签筛选、按模型置信度排序（优先审核模型最不确定的样本）。提供一键通过全部。审阅完成后点击确认进入下一轮，触发模型训练。管理员可在项目配置中关闭审阅阶段，标注完成直接进入训练。

### 标记对象渲染系统

标注对象不局限于纯文本。通过标记类型插件定义字段结构和渲染方式。内置纯文本类型，示例有爬虫评论类型（含用户名、头像、时间、内容、点赞数等字段）。管理员可上传 Python 脚本自定义类型，后续支持拖拽式 GUI 设计。每种标记类型定义四种呈现视图：专家标注视图（完整 HTML 页面）、列表浏览视图（一行摘要）、模型输入视图（送入编码器的文本拼接）、导出视图（CSV 行或 JSON 对象格式）。同一个标记对象在不同场景自动切换对应视图。

### 数据导入与字段映射

当前版本支持上传 txt 文件，一行一条样本。架构已为后续扩展预留：CSV 导入时可图形化映射 CSV 列名到内容表字段名，支持改名映射和忽略不需要的列，必填字段必须有映射；数据库直连时自动全部字段映射；多媒体包上传后自动解压，路径存为字段值，渲染脚本将路径转为图片或视频标签。导入数据时先创建数据集记录，再异步批量计算向量。

### 弱监督

管理员预先编写标注规则，比如关键词匹配、正则表达式。系统自动用这些规则给样本打弱标签，逻辑规则引擎自动修正冲突、丢弃不可靠的弱标签。干净的弱标签直接参与模型训练，在标注早期人工样本极少时帮模型快速起步，大幅降低所需人工量。弱监督作为算法的一个可选组件，管理员创建项目时选择是否启用。

### 覆盖可视化

高维嵌入向量降维到二维平面展示。已标注样本显示为彩色圆点并带有覆盖圆（半径代表该点能可靠覆盖的范围），覆盖区域半透明着色，未覆盖区域保持原色。不同类别用不同颜色区分。支持缩放查看局部细节，拖动轮次滑块观看覆盖从稀疏到密布的演变过程，悬停任意点显示摘要信息。这是论文核心概念的可视化呈现，领导一眼看懂项目进展。

### 仪表盘与统计

轮次趋势表展示每轮覆盖率、模型精度、人机耗时、专家活跃比例、人机一致率。累计人机耗时比直观反映效率。弱监督启用时单独展示弱标签覆盖率、准确率、节省的人力百分比、收敛加速对比（有弱监督 vs 无弱监督所需轮次和耗时）、各标注规则的贡献分布、被修正和丢弃的弱标签数量。专家维度展示每位专家的人机一致性、与其他专家间的一致性、平均标注耗时。管理员可拉取最近若干条不一致标注记录用于复盘，可按全局或按特定专家筛选。

### 权限管理

两层权限体系。系统角色控制能访问哪些 API 模块——内置观察员（只读）、标注专家（抢单提交）、审阅员（审核裁定）、项目管理员（创建项目分配成员）、超级管理员（全部权限）。管理员可基于内置角色复制并增减 API 白名单，创建自定义系统角色。项目角色控制具体项目内的操作权限——Owner、Manager、Annotator、Reviewer、Observer 五级。项目分为公开和私有，公开项目所有登录用户可见，私有项目只有被显式添加的成员可见。同一用户在不同项目中可担任不同角色。新注册用户默认只有观察员系统角色，无任何项目成员关系，只能浏览公开项目。

### 版本与授权

私有部署，通过 license 文件区分版本。免费版永久有效、功能完整，但整个部署实例中所有项目样本总数不超过两百条、项目数不超过十个、不可删除项目。导入时超出部分自动截断并提示。付费版按订阅周期授权，取消所有限制，提供版本更新和技术支持。订阅到期后标注和创建功能锁定，但历史数据的查看和导出不受影响。license 文件带签名防篡改，付费版支持重签续期。

### 审计日志

所有关键操作记录到审计日志：登录、创建项目、导入数据、启动项目、抢单、提交标注、放弃任务、审阅修改、裁定、导出、修改角色权限、禁用用户等。记录内容包含操作者、操作类型、操作对象、变更详情、客户端 IP。管理员可按用户、项目、操作类型、时间范围检索。普通用户不可见审计日志。

### 用户事件日志

面向所有项目参与者的项目时间线。展示里程碑事件：数据导入完成、第 N 轮开始、第 N 轮标注完成、第 N 轮审阅通过、模型训练完成及精度、覆盖率更新、项目达标完成、管理员暂停或恢复、降级事件。与审计日志互补——审计记谁做了什么操作，事件记项目发生了什么。

### 开发日志

传统文件日志，按天滚动，按模块和级别分类。DEBUG 级别记录详细调试信息，INFO 记录轮次开始结束等关键节点，WARN 记录降级重试等非致命异常，ERROR 记录需人工介入的错误。ERROR 级别单独输出一份文件保留更长时间。

### 错误处理

三级策略：操作失败时先自动重试数次，重试仍失败则尝试降级运行，无法降级则终止并返回错误码。每个组件有预定义的降级方案——编码器不可用时跳过该样本继续、模型训练失败时降级为 CPU 训练或使用上轮模型、弱监督不可用时退化为纯人工模式、覆盖率计算异常时用上轮近似值。所有错误携带结构化错误码，格式为模块-子模块加序号，用户反馈错误码即可快速定位问题模块。

### 算法插件

算法名称是用户看到的唯一概念，创建项目时从下拉列表选择。内部由多个可替换组件装配完成：编码器负责将文本转为向量、选样策略负责从无标注对象中选出本轮给专家标的样本、模型训练器负责训练分类器并预测、弱监督引擎可选。平台内置论文原版 SALT 实现，后续管理员可注册自定义算法。编码器支持注册多个。每轮训练产出一个模型版本，支持版本管理和回溯，管理员可手动切换使用历史模型。向量只在导入后计算一次入库，后续轮次直接读取，不重复计算。

### 数据导出

项目完成后，管理员或数据导出员可导出标注结果。支持 CSV、JSON、TXT 三种格式，每行包含原始内容、最终标签、标签来源（人工或机器）、置信度。可选择只导出机器自动标注的部分、或导出全部。导出操作记入审计日志。
