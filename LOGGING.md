# 日志设计

两层日志：开发日志写文件，用户日志写数据库。

---

## 1. 开发日志（文件）

传统日志，给程序员排查问题。

### 1.1 格式

```
2026-06-26 14:32:15.123 [INFO] [salt.engine.selector] project=proj-001 round=3 selected=40 coverage=0.58
2026-06-26 14:32:18.456 [ERROR] [salt.engine.trainer] project=proj-001 round=3 loss=NaN step=1200
2026-06-26 14:32:18.457 [WARN] [salt.engine.trainer] project=proj-001 round=3 degraded to cpu training
```

### 1.2 级别

| 级别 | 用途 |
|------|------|
| DEBUG | 详细的调试信息，默认不输出 |
| INFO | 关键节点：轮次开始/结束、训练完成、覆盖率达标 |
| WARN | 降级、重试、资源不足等非致命异常 |
| ERROR | 组件失败、数据异常、需人工介入的错误 |

### 1.3 模块命名

```
salt.engine.selector     # 选样
salt.engine.trainer      # 训练
salt.engine.encoder      # 编码
salt.engine.coverage     # 覆盖率
salt.engine.weaksupervise# 弱监督
salt.api.tasks           # 标注任务 API
salt.api.projects        # 项目 API
salt.db                  # 数据库
```

### 1.4 滚动策略

- 按天滚动：`salt_2026-06-26.log`
- 保留 30 天
- ERROR 级别单独输出一份 `salt_error.log`，保留 90 天

---

## 2. 用户日志（数据库）

给用户看的项目事件时间线。与审计日志（`AUDIT.md`，记录谁做了什么）互补——用户日志侧重**系统事件**。

### 2.1 表结构

```
project_events:
  id            BIGSERIAL PK
  project_id    UUID NOT NULL
  round         int | null              # 发生在第几轮
  event_type    string                  # 事件类型
  message       string                  # 人类可读描述
  detail        json | null             # 附加数据
  created_at    timestamp DEFAULT NOW()
```

### 2.2 事件类型

| event_type | message 示例 | 触发时机 |
|-----------|-------------|---------|
| `import_done` | "数据导入完成，共 10,000 条样本" | 导入 + embedding 全部完成 |
| `round_start` | "第 3 轮开始，选中 40 条样本" | 每轮开始 |
| `round_annotated` | "第 3 轮标注完成，专家标注 40 条" | 本轮全部提交 |
| `round_reviewed` | "第 3 轮审阅通过" | 管理员确认审阅 |
| `round_trained` | "第 3 轮模型训练完成，精度 0.85" | 训练完成 |
| `coverage_update` | "覆盖率 58%，风险差距 0.12" | 覆盖率计算完成 |
| `project_complete` | "项目达到目标精度，共 5 轮，标注 320 条" | 项目完成 |
| `project_paused` | "管理员暂停了项目" | 暂停 |
| `project_resumed` | "项目已恢复" | 恢复 |
| `model_reused` | "从项目「评论分类」克隆了模型，热启动" | 克隆项目时复用模型 |
| `degraded` | "弱监督引擎不可用，已切换为纯人工模式" | 降级事件 |
| `error` | "模型训练失败，错误码 MD-003，将用上轮模型继续" | 错误 + 降级 |

### 2.3 展示

```
┌──────────────────────────────────────────────────────┐
│  项目事件  │  评论情感分类                             │
├──────────────────────────────────────────────────────┤
│                                                      │
│  06-26 14:32  项目达到目标精度，共 5 轮，标注 320 条    │
│  06-26 14:30  第 5 轮模型训练完成，精度 0.91           │
│  06-26 14:25  第 5 轮审阅通过                          │
│  06-26 14:10  第 5 轮标注完成，专家标注 28 条           │
│  06-26 14:05  第 5 轮开始，选中 28 条样本              │
│  06-26 13:50  覆盖率 81%，风险差距 0.08               │
│  06-26 13:48  第 4 轮模型训练完成，精度 0.85           │
│  ...                                                 │
│  06-26 10:00  数据导入完成，共 10,000 条样本            │
│                                                      │
│              [仅看错误]  [仅看里程碑]                  │
└──────────────────────────────────────────────────────┘
```

### 2.4 与审计日志的区别

| | 审计日志 (audit_logs) | 用户事件 (project_events) |
|------|---------------------|-------------------------|
| 受众 | 管理员、安全审计 | 所有项目参与者 |
| 内容 | 谁做了什么操作 | 项目发生了什么事件 |
| 粒度 | 每次点击都记 | 仅记里程碑节点 |
| 示例 | "张三抢单 #42" | "第 3 轮标注完成" |
| 查询 | 按用户/操作类型筛选 | 按项目/轮次浏览时间线 |
