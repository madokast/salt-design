SALT 标注平台

基于论文 SALT (Selective Annotation for Labeling and Training) 的企业级数据标注平台设计文档。用最少的专家标注量，达到有理论保证的标注精度。

系统总览

多角色在线协作的数据标注平台，SALT 算法驱动的 HITL（人机协同）迭代标注

SALT

  Selective Annotation for Labeling and Training（选择性标注：只标该标的部分，剩下的机器自动完成）。

  • 目标 — 给大规模无标注数据集打标签。人工标注尽可能少，理论保证自动标注精度
  • 传统方法 — 雇人一条条标，一万条文本需上千人时。专家更贵，成本不可持续
  • SALT 方法 — 人只标一小部分，机器学完后标剩下的
  • 人机循环 — 模型选最有价值的样本 → 专家在线抢单 → 模型学习 → 检查覆盖率；达标即停
  • 效果 — 标注精度提升最高 72.4%，人力节省最高 75.7%，收敛时间减少 44%+

单轮 SALT 循环

标注协作循环

一张任务表承载全部状态，乐观锁串行化，专家在线抢单

抢单机制

  • 任务表每条记录三种状态：待抢 → 标记中 → 已提交
  • 专家优先从待抢池中取，CAS 更新为标记中
  • 待抢池空则从其他专家的标记中直接抢走
  • 所有并发冲突通过乐观锁（version）解决，不长期持有资源
  • 提交或放弃后可抢下一条，一次只持有一条

核心特性

逐一作答 一次一条

专家每次只持有一条任务，必须提交或放弃后，才能抢下一条。不允许多条堆积

乐观锁冲突 version

提交时校验 version，被他人抢先则提示刷新

机器辅助 可选

模型先作答，专家标注时可看到机器推荐

动态人力 可选

机器高置信 → 1 人复核；低置信 → 2 人交叉验证

多人裁定 可选

人-人/人-机不一致 → 管理员最终裁决

质量抽检 可选

随机 p% 样本强制双人标注，监控专家一致性

数据模型

双库分离——结构数据高频备份、标注数据独立扩容，职责清晰互不干扰

为什么分离？ 用户、权限、项目配置等结构数据量小（MB 级）但至关重要，需要高频备份和高可用。标注数据随项目规模线性增长（GB 级），按项目动态建表，备份频率低，可独立扩容或分片。两者物理隔离后，标注库的批量读写不会拖慢主库的权限校验和页面响应，删项目只需 DROP 标注库的表。

处理流程

LF（标注函数）

管理员编写的自动标注规则。输入 = 标记对象的 model 视图文本，输出 = 标签或弃权（None）。四种类型，每条 LF 独立逐条执行，不需看其他对象。

关键字 keyword

包含"涨停""央行""GDP" → 输出「财经」。简单快速，覆盖面广

正则 regex

匹配股票代码 \d{6}\.(SZ|SH) → 输出「财经」。结构化模式

外部知识库 kb

查公司名表：文本中的实体命中科技公司表 → 输出「科技」

Python 脚本 python

任意逻辑：文本长度 > 200 且含"公告" → 输出「财经」

示例

文本：「苹果公司发布财报股价大涨，NBA湖人队晋级季后赛」

LF匹配输出
LF_keyword_money"股价"财经
LF_keyword_sports"NBA"体育
LF_entity_org"苹果"科技
LF_regex_stock无匹配弃权

三条 LF 给出三个不同标签——冲突。交给融合模型 P。

融合模型 P

轻量概率图模型（非深度学习），与 Snorkel 原理相同。从 LF 间的一致/矛盾模式推断每条 LF 的类别条件错误率，综合给出软标签。不需要任何人工标注——仅凭 LF 输出模式自我校准。

需要完整 Λ 矩阵（全部对象 × 全部 LF 的输出）才能工作，不能逐条处理。权重每次执行重新估计，不固化。软标签仅模块内部流转，不对外输出。

LRR（标签修正规则）

确定性逻辑规则，修正 P 也无法解决的错误模式。形式为 IF-THEN：前提条件命中 → 强制修正标签或丢弃。

冲突裁决

多条 LF 矛盾时选更可靠的：
IF LF_sports=体育 ∧ LF_org=科技 THEN 取体育
（历史数据中体育关键词更可靠）

证据传播

利用样本间关系传播标签：
IF 已标样本 y=体育 ∧ 同主题 THEN 标签=体育

噪声过滤

丢弃低置信度标签：
IF LF 输出科技 ∧ Pr(科技)<0.15 THEN 丢弃

冲突解决

多条 LRR 同时命中 → 按加权打分排序（置信度×0.5 + 特异性×0.3 + 一致性×0.2），差距太小则保守弃权

完整示例

文本：「苹果公司发布财报股价大涨，NBA湖人队晋级」

1. LF 原始输出: 财经 / 体育 / 科技 / 弃权（冲突）
2. P 软标签: 财经 0.42 · 体育 0.47 · 科技 0.09
3. LRR 规则1 命中: IF LF_sports=体育 ∧ LF_org=科技 → 取体育
4. LRR 规则2 命中: IF Pr(科技)<0.15 → 丢弃科技
5. 最终输出: 标签=体育, 置信度=0.47
