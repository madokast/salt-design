# 算法总纲

按用户操作流程出现的顺序列出所有需要编码的算法。不是理论推导，是输入输出明确的实现单元。

---

## 1. embedding 批量计算

- **时机**：上传数据后，项目启动前
- **输入**：所有标记对象的文本（由 anno_type 的 model 视图拼装）
- **输出**：每条对象的 768 维浮点向量，入库
- **说明**：一次性计算，后续轮次直接读库，不重算

## 2. 初始样本选取

- **时机**：项目首次启动，模型尚未训练
- **输入**：全部未标注对象
- **输出**：随机选取 `initial_labeled` 条对象 ID
- **说明**：无模型信息可用，纯随机，只执行一次

## 3. 模型训练

- **时机**：每轮标注 + 审阅确认后
- **输入**：已标注对象（embedding + final_label）、标签集、上轮模型（热启动）或空（冷启动）
- **输出**：训练好的模型文件 + 指标（accuracy、loss）
- **说明**：MLP 分类器，交叉熵损失，每轮可冷启动或热启动

## 4. 模型预测

- **时机**：模型训练后，若启用机器作答
- **输入**：当前模型 + 本轮未标注对象的 embedding
- **输出**：每条对象的预测标签 + 置信度（max_prob）
- **说明**：结果写入 MACHINE task，用于标注界面推荐和置信度驱动人力分配

## 5. 覆盖半径计算

- **时机**：模型训练后
- **输入**：当前模型 \( \theta \)、已标注对象 \( S \)、精度容忍度 \( \zeta \)、类别数 \( C \)
- **输出**：每个标注对象 \( x_i \) 的覆盖半径 \( \delta_{x_i} \)
- **说明**：论文公式 (3)，依赖谱范数 \( \Gamma_\theta \) 和局部 Lipschitz 常数 \( \lambda \)。平坦区域半径大，陡峭区域半径小。用于覆盖率检查和可视化

## 6. 覆盖率检查

- **时机**：覆盖半径计算后
- **输入**：全部对象、每个标注对象的覆盖半径
- **输出**：覆盖率、风险差距、是否达标
- **说明**：论文 Theorem 3 + Corollary 4。判断已标注对象的覆盖球是否覆盖了全部数据

## 7. 下一轮选样

- **时机**：覆盖率检查未达标
- **输入**：全部对象 + embedding、已标注对象 + 覆盖半径、当前模型、未覆盖对象、配额 `bgt`、超参数 \( \alpha \)
- **输出**：下一轮需要标注的对象 ID 列表（`bgt` 条）
- **说明**：论文 Algorithm 1 + 公式 (5)。贪心最大化覆盖增益，评分 = 覆盖新对象数 + 自适应权重 × 预测不确定性。早期重覆盖扩展，后期重边界细化

## 8. 弱监督（若启用）

### 8a. 弱标签生成

- **时机**：选样后、生成 HUMAN task 前
- **输入**：本轮选中对象、所有 labeling functions
- **输出**：每条对象的弱标签（含置信度）

### 8b. LRR 规则修正

- **时机**：弱标签生成后
- **输入**：弱标签列表、人工标注列表（校准用）、LRR 规则集
- **输出**：修正后的弱标签（冲突已裁决、不可靠已丢弃）
- **说明**：论文第 6 章。规则优先于 LF 投票，不可靠的弱标签不参与训练

## 9. 降维与归一化

- **时机**：数据导入后（一次性）
- **输入**：全部对象的 embedding（768 维）
- **输出**：每条对象的 2D 坐标 (x, y)，归一化到 (0,1)×(0,1)，入库
- **说明**：t-SNE 或 UMAP。embedding 固定 → 投影固定。覆盖可视化每轮只需重新着色和更新覆盖圆半径，坐标不变
